Принципы ООП

  • Абстрагирование — выделение существенных характеристик объекта, игнорирование несущественных.
  • Инкапсуляция — сокрытие внутреннего состояния и предоставление интерфейса через public/private.
  • Наследование — создание нового класса на основе существующего, переиспользование кода.
  • Полиморфизм — единый интерфейс для разных типов (виртуальные функции).
  1. Инкапсуляция в C++Доступ к членам класса ограничивается спецификаторами: private (только внутри класса), protected (внутри и в наследниках), public (везде). Данные обычно делают private, а доступ через public методы (геттеры/сеттеры).
  2. Наследование в C++Синтаксис: class Derived : public Base { };. Типы наследования: public (сохраняет уровни доступа), protected (сводит public к protected), private (сводит всё к private). Конструктор производного класса вызывает конструктор базового.
  3. Полиморфизм в C++Реализуется через виртуальные функции (virtual). Базовый указатель может вызывать переопределённые методы производного класса. Чисто виртуальная функция: virtual void f() = 0; делает класс абстрактным.
